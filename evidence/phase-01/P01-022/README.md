## Failures, skips, and limitations

- The first checker draft reused its strict JSON reader for generated Markdown and failed immediately at character zero. Text and JSON readers were separated before the artifact commit.
- Scope review found that a MongoDB-only closed-world policy was insufficient because the project also names Redis-like compatibility. The final matrix adds explicit native and Redis closed-world rules plus 33 Redis unsupported categories.
- No finalized required check failed, and no matrix, semantic, or differential row was skipped.
- `reference_only` does not mean product implementation. The storage/query engine remains absent; 56 contract-only rows are normative design without successful engine execution.
- Limit rows use compact below/at/above oracle actions rather than allocating maximum-size production objects. Error rows validate canonical metadata synthetically rather than injecting every future subsystem fault.
- The 16 MongoDB rows are hand-selected query fixtures over six documents. They are not generated breadth, adapter/protocol behavior, or a user-facing support claim.
- Redis has no pinned upstream, protocol harness, or differential case at this milestone; all Redis behavior is unsupported.
- The matrix must be versioned and regenerated as engine, protocol, SDK, adapter, migration, or later-release work changes classifications. It cannot authorize future support without executable evidence.
